Criando uma SideBar com React

May 27, 2020

Recentemente precisei implementar um menu de navegação para um PWA. Como estava fazendo do zero, sem nenhuma biblioteca de UI como Material ou Semantic, fui pesquisar alguns exemplos pra me basear. Porém, depois de alguma pesquisa, não encontrei nenhum exemplo bom e simples o suficiente para o que estava procurando. Daí a motivação de escrever este post detalhando como eu implementei uma SideBar responsiva com React e espero que possa ser útil para alguém :)

[Spoiler] Segue aqui o componente pronto e funcional:

Criando o projeto

Vamos criar um projeto React com o famoso create-react-app para fins práticos.

Para criar o projeto, execute:

$ npx create-react-app sidebar-app

npx é um package runner que vem junto com o Node. Então tenha uma versão atualizada do Node instalado.

Entre na pasta sidebar-app e pronto! Temos uma base de um projeto React pronta pra ser customizada.

É interessante remover alguns arquivos que já vem no create-react-app por padrão, como os logos na pasta public e o arquivo App.css na pasta src. Não esqueça de fazer as correções necessárias no App.js e index.html.

Agora inicie o servidor de desenvolvimento do Webpack com o comando:

$ yarn start

Abra o navegador na rota localhost:3000 e você deverá ter uma tela em branco.

Criando o componente SideBar

Primeiro, crie uma pasta components dentro de src e depois uma pasta SideBar.

O path fica src -> components -> SideBar

Dentro de SideBar fica a lógica e estilo do nosso componente.

Começando pela lógica, crie o arquivo SideBar.js dentro de SideBar com a estrutura padrão de um componente React:

// src/SideBar/SideBar.js
import React from "react";
const SideBar = () => {
  return (
    <nav>
      <ul></ul>
    </nav>
  );
};
export default SideBar;

Pensando na estrutura do componente, nossa Sidebar vai ser basicamente um <nav> com uma lista de itens. Essa lista de itens na verdade será uma lista de um outro componente, o SidebarItem…mas nós vamos chegar lá.

Agora vamos estilizar o componente. Aqui, seguirei a filosofia de mobile first, isto é, o estilo sempre é aplicado pensando na visualização mobile e então ajustado pra visualização desktop. E vou utilizar variáveis CSS por ser um recurso poderoso do CSS e que hoje já possui um ótimo suporte nos principais navegadores. As variáveis serão importantes para armazenar, por exemplo, cores e espaçamento dos nossos componentes.

Para as variáveis ficarem disponíveis por toda a aplicação, crie-as no arquivo src/index.css (já existente).

/* src/index.css */
:root {
  --gray-dark: #37474f;
  --sidebar-width: 80px;
  --sidebar-height: 60px;
  --sidebar-z-index: 1000;
}

Possivelmente já terá um código existente no arquivo, pode deixá-lo aí ou apagar se preferir.

A propriedade :root faz com que todas as variáveis definidas ali fiquem disponíveis em todo o escopo html, ou seja, em toda a aplicação.

Crie um arquivo SideBar.css no mesmo nível do SideBar.js com o seguinte código:

/* src/components/SideBar/SideBar.css */
.sideBarContainer {
  background-color: var(--gray-dark);
  width: 100vw;
  position: fixed;
  bottom: 0;
  z-index: var(--sidebar-z-index);
  box-shadow: var(--sidebar-shadow-mobile);
}

Agora precisamos importá-lo no SideBar.js e atribuir essa classe a tag <nav>:

// src/components/SideBar/SideBar.js
import React from "react";
import "./SideBar.css";
const SideBar = () => {
  return (
    <nav className="sideBarContainer">
      <ul></ul>
    </nav>
  );
};
export default SideBar;

E para visualizar o componente, importe-o no App.js:

// src/App.js
import React from "react";
import SideBar from "./components/SideBar/SideBar";
function App() {
  return (
    <div className="App">
      <SideBar />
    </div>
  );
}
export default App;

Perceba que temos um detalhe pra resolver. Na visualização desktop temos que fazer com que o componente apareça na vertical, pois por enquanto está na horizontal (igual o mobile). Para isso, vamos usar media query, uma propriedade responsiva do CSS. Voltando então ao arquivo SideBar.css, adicione o seguinte bloco de código:

/* src/components/SideBar/SideBar.css */
@media only screen and (min-width: 1200px) {
  .sideBarContainer {
    width: var(--sidebar-width);
    height: 100vh;
    box-shadow: var(--sidebar-shadow-desktop);
    transition: 300ms ease;
  }
  .sideBarContainer:hover {
    width: 250px;
  }
}

Nosso componente tá tomando forma! \o/ Um detalhe bacana de adicionar, e que é bastante comum em componentes do tipo, seria um overlay no evento de hover, isto é, ao abrir a SideBar o resto da tela fica com uma camada mais escura sobre, destacando a SideBar.

Isso é simples de fazer. Para adicionar o overlay, crie uma tag <div> no componente SideBar:

// src/components/SideBar/SideBar.js
const SideBar = () => {
  return (
    <>
      <div className="sideBarOverlay"></div>
      <nav className="sideBarContainer">
        <ul></ul>
      </nav>
    </>
  );
};

Perceba que tivemos que colocar tudo dentro de uma tag <>. Esse é um componente default do React que eles chamam de Fragment. É útil quando não se quer colocar tudo dentro de um <div>, já que o React não aceita que o componente tenha 2 ou mais tags no mesmo nível.

E no arquivo SideBar.css adicione a classe:

/* src/components/SideBar/SideBar.css */
@media only screen and (min-width: 1200px) {
  .sideBarOverlay {
    position: fixed;
    height: 100vh;
    width: 100%;
    background: rgba(0, 0, 0, 0.3);
    z-index: var(--sidebar-z-index);
    transition: 200ms ease;
  }
}

Sucesso! Temos um overlay. Perceba, porém, que dessa maneira o overlay é sempre aplicado, ao invés de aplicarmos ele apenas no hover da SideBar. Para isso, tal tratamento tem que ser feito no próprio componente, pois o CSS não consegue lidar com a lógica de “se há hover nesse elemento aplique um estilo em outro elemento”. Esse tipo de tratamento tem que ser feito com JavaScript.

Uma coisa bem poderosa que as versões mais recentes do React (a partir da versão 16.8 pra ser mais exato) adicionou foram os hooks. Os hooks permitem que a gente reutilize lógicas comuns como funções.

Aqui, vou criar um hook pro evento de hover. É um hook que só vai ser utilizado no componente SideBar mas que poderia tranquilamente ser usado em qualquer outro componente da aplicação.

Crie então uma pasta hooks/ dentro de src/ e adicione o arquivo useHover.js com o seguinte código:

// src/hooks/useHover.js
import { useRef, useState, useEffect } from "react";
const useHover = () => {
  const [value, setValue] = useState(false);
  const ref = useRef(null);
  const handleMouseOver = () => setValue(true);
  const handleMouseOut = () => setValue(false);
  useEffect(() => {
    const node = ref.current;
    if (node) {
      node.addEventListener("mouseover", handleMouseOver);
      node.addEventListener("mouseout", handleMouseOut);
      return () => {
        node.removeEventListener("mouseover", handleMouseOver);
        node.removeEventListener("mouseout", handleMouseOut);
      };
    }
  }, []);
  return [ref, value];
};
export default useHover;

Esse hook vai nos retornar um valor e uma referência. O valor é um booleano que tem valor true no hover e falso caso contrário. A referência é o elemento que queremos usar como referência pro evento de hover. Vamos usá-lo então no nosso componente. Para isso, basta importá-lo e aplicar a lógica de hover na tag do overlay:

// src/components/SideBar/SideBar.js
...
import useHover from "../../hooks/useHover";
const SideBar = () => {
  const [hoverRef, isHovered] = useHover();
  ...
  return (
    <>
      <div className={`${isHovered ? "sideBarOverlay" : null}`}></div>
      <nav className="sideBarContainer" ref={hoverRef}>
      ...

Se isHovered nos retorna true, então aplique o estilo de overlay, senão não aplique nenhum estilo.

Considere nosso componente SideBar pronto. “Mas e no mobile?! O componente nem altura tem…” Calma jovem, nosso componente vai ganhar altura assim que colocarmos coisas nele. E essas coisas são instâncias do componente SideBarItem.

Criando o componente SideBarItem

Precisamos agora criar os elementos da lista contida no componente SideBar. Eu vou chamar esses elementos de SideBarItem e eles também serão um componente React.

Esse componente, basicamente, será um <li> com um ícone, um texto (na visualização desktop) e um link. Crie então a pasta SideBarItem (dentro de components) com o arquivo SideBarItem.js e este com a seguinte estrutura:

import React from "react";
const SideBarItem = ({ icon, text, to }) => {
  return (
    <li className="itemContainer">
      <a className="itemLink" href={to}>
        <div>
          <span>
            <i className={icon}></i>
          </span>
        </div>
        <div className="itemText">
          <span>{text}</span>
        </div>
      </a>
    </li>
  );
};
export default SideBarItem;

Então nosso componente recebe como props um ícone, um text e um link. Para os ícones, utilizaremos a biblioteca Font Awesome, que nos disponibiliza alguns ícones gratuitamente.

Existem várias maneiras de usar o Font Awewome. A maneira que eu achei mais conveniente, pois envolve menos código, foi a de acessar o kit de ícones via CDN. Porém, esta maneira pode ser um pouco chata por necessitar criar uma conta no site do Font Awesome. Caso queira seguir dessa maneira, acesse aqui pra se cadastrar. Uma outra maneira é instalar os pacotes oficiais yarn ou npm a partir deste repositório.

Seguindo então com o CDN, após o cadastro o Font Awesome irá te fornecer um link pra ser referenciado no arquivo public/index.html. Apenas coloque o link neste arquivo como:

<head>
  ...
  <script src="{link}" crossorigin="anonymous"></script>
</head>

Feito isso, agora podemos usar a tag <i> com a classe associada a algum ícone do Font Awesome para gerarmos um ícone no nosso componente.

Esta classe é a props icon do componente SideBarItem

Precisamos agora estilizar minimamente nosso componente. Por enquanto ele tem as classes itemContainer, itemLink e itemText. Para isso, crie um arquivo SideBarItem.css na pasta do componente com o seguinte conteúdo:

.itemLink {
  display: flex;
  align-items: center;
  justify-content: space-around;
  height: var(--sidebar-item-height);
  border-radius: 4px;
  transition: 0.2s ease;
  margin: 8px;
  text-decoration: none;
  font-weight: 600;
  color: #fff;
}
.itemLink:hover {
  background-color: var(--gray-darkest);
}
.itemText {
  display: none;
}

As propriedades text-decoration, font-weight e color serão aplicadas ao texto que aparecerá no hover.

Basicamente alinhamos os itens (ícone e texto) na horizontal e setamos uma altura (pensando na visualização mobile).

Não esqueça de importar os estilos no componente: import "./SideBarItem.css";

Perceba também que adicionamos mais algumas variáveis CSS. Portanto, atualize o arquivo index.css com elas:

/* src/index.css */
:root {
  ...
  --gray-darkest: #222b2f;
  ...
  --sidebar-item-height: 50px;
  --sidebar-item-height-desktop: 70px;

Vamos testar então o componente e ver se esses estilos precisam de algum ajuste ou não. A SideBar desse exemplo possui 4 itens, mas pode ter mais ou menos de acordo com a necessidade. Para ficar uma estrutura flexível, crie um arquivo pra armazenar esses itens. No meu caso, vou criar na mesma pasta do componente SideBar:

// src/components/SideBar/items.js
const items = [
  { id: 1, text: "Home", icon: "fas fa-home", to: "/" },
  {
    id: 2,
    text: "Localização",
    icon: "fas fa-map-marked-alt",
    to: "/localizacao",
  },
  { id: 3, text: "Contato", icon: "fas fa-paper-plane", to: "/contato" },
  { id: 4, text: "Entrar", icon: "fa fa-sign-in", to: "/login" },
];
export default items;

Agora importe e renderize esses itens no componente SideBar:

// src/components/SideBar/SideBar.js
...
import SideBarItem from "../SideBarItem/SideBarItem";
import items from "./items";
const SideBar = () => {
  ...
  <ul>
    {items.map((item) => {
      return (
        <SideBarItem
          key={item.id}
          text={item.text}
          icon={item.icon}
          to={item.to}
        />
      );
    })}
  </ul>

Tá chegando! Agora precisamos fazer uns ajustes. Primeiro, perceba que os itens estão na vertical na visualização mobile. Além disso, a lista de ícones está com o estilo padrão da tag <ul>. Vamos arrumar isso. No arquivo SideBar.css, adicione os estilos:

/* src/components/SideBar/SideBar.css */
.sideBarList {
  display: flex;
  justify-content: space-around;
  list-style: none;
  padding: 0;
  margin: 0;
}
@media only screen and (min-width: 1200px) {
  ...
  .sideBarList {
    display: flex;
    flex-direction: column;
    margin-top: var(--spacing-medium);
  }

E adicione a classe na tag <ul>:

// src/components/SideBar/SideBar.js
...
<ul className="sideBarList">
  {items.map((item) => {
...

Parece quase pronto agora. Um detalhe importante, é que a área clicável/tocável do ícone na visualização mobile está muito pequena. Para aumentar essa área, apenas faça com que o container do <li> ocupe todo o espaço disponível. Isso é facilmente alcançado adicionando:

/* src/components/SideBarItem/SideBarItem.css */
.itemContainer {
  flex: 1;
}
...

E por último (ufa!), precisamos adicionar o texto na visualização desktop e no evento de hover. Um jeito possível de fazer isso é adicionar dinamicamente uma classe CSS no evento de hover a partir do nosso hook useHover. O componente pai SideBar já utiliza ele, e como queremos que o texto de todos os itens apareçam ao mesmo tempo, passaremos esse evento como uma props pros componentes SideBarItem.

Então no componente SideBar:

// src/components/SideBar/SideBar.js
<SideBarItem
  ...
  hover={isHovered}
/>

Não esqueça de adicionar a props no componente filho SideBarItem

Criando a classe CSS:

/* src/components/SideBarItem/SideBarItem.css */
@media only screen and (min-width: 1200px) {
  .itemTextHover {
    display: block;
    width: 50%;
  }
}

A propriedade width aqui serve pra deixar os textos e ícones alinhados entre si

E agora adicione a classe dinamicamente:

// src/components/SideBarItem/SideBarItem.js
...
  <div className={`itemText ${hover ? `itemTextHover` : null}`}>
    <span>{text}</span>
  </div>
...

E pronto! \o/ Temos uma SideBar no desktop e uma AppBar no mobile, tudo no mesmo componente :)
